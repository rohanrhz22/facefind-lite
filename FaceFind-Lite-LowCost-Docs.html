FaceFind Lite

The same "upload photos → find a person" face-recognition app, re-engineered to run at near-zero cost using free tiers, open-source models, and a browser-first design. Start at $0/month and scale only when you earn revenue.

💸 $0 to start 🧠 Open-source models (free) 🌐 Runs in-browser (optional) ☁️ Free-tier hosting 🎯 ~99% accuracy

01Overview

FaceFind Lite delivers the identical feature — upload a gallery and one reference selfie, get back every photo of that person — but optimizes every layer for cost. It does this with three big levers: (1) use free open-source face models (zero per-call fees), (2) run on free hosting tiers / a single cheap box, and (3) optionally push the AI into the user's browser so you pay almost nothing for compute.

03How It Works (same core)

🟦 1. Detect

Find every face in each photo and crop + align it.

🧬 2. Embed

Turn each face into a numeric "fingerprint" vector (128 or 512 numbers).

📐 3. Compare

Cosine similarity between the query face and stored faces → above threshold = match.

Same maths, cheaper plumbing. The recognition logic is identical to a premium build — we only change where it runs and what stores the vectors so the bill stays at zero.

09Data Model (SQLite — zero cost)

-- photos table
CREATE TABLE photo (
  id          INTEGER PRIMARY KEY,
  filename    TEXT,
  data        BLOB,          -- store image (or path to R2/disk)
  created_at  TEXT
);

-- one row per detected face
CREATE TABLE face (
  id          INTEGER PRIMARY KEY,
  photo_id    INTEGER REFERENCES photo(id),
  bbox        TEXT,          -- JSON [x,y,w,h]
  embedding   BLOB           -- 512 float32 packed bytes (np.tobytes)
);

For small/medium galleries, brute-force cosine in NumPy over all stored vectors is fast enough (thousands of faces in milliseconds) — no vector database needed. Add FAISS only past ~100k faces.

10Reference Code

Mode B — Server (FastAPI + InsightFace + SQLite + NumPy)

import numpy as np, sqlite3, io
from insightface.app import FaceAnalysis
from PIL import Image

model = FaceAnalysis(name="buffalo_l")     # free, ~99% accuracy
model.prepare(ctx_id=-1, det_size=(640, 640))   # CPU

def embed_faces(img_bgr):
    return [(f.bbox.tolist(), f.normed_embedding.astype("float32"))
            for f in model.get(img_bgr)]

def index_photo(db, photo_id, img_bgr):
    for bbox, emb in embed_faces(img_bgr):
        db.execute("INSERT INTO face(photo_id,bbox,embedding) VALUES (?,?,?)",
                   (photo_id, str(bbox), emb.tobytes()))
    db.commit()

def search(db, query_img_bgr, threshold=0.5):
    faces = embed_faces(query_img_bgr)
    if not faces: return []
    q = faces[0][1]                                  # query vector (already L2-normalized)
    rows = db.execute("SELECT photo_id, embedding FROM face").fetchall()
    best = {}
    for photo_id, emb_bytes in rows:
        v = np.frombuffer(emb_bytes, dtype="float32")
        sim = float(np.dot(q, v))                    # cosine (vectors normalized)
        if sim >= threshold and sim > best.get(photo_id, 0):
            best[photo_id] = sim
    return sorted(({"photo_id": p, "similarity": s} for p, s in best.items()),
                  key=lambda r: r["similarity"], reverse=True)

Mode A — Browser-only (face-api.js, $0 backend)

import * as faceapi from 'face-api.js';

// Load free models once (host the weights as static files)
await faceapi.nets.ssdMobilenetv1.loadFromUri('/models');
await faceapi.nets.faceLandmark68Net.loadFromUri('/models');
await faceapi.nets.faceRecognitionNet.loadFromUri('/models');

// Build descriptors (128-d) for every gallery image — all in the browser
async function describe(imgEl) {
  return faceapi.detectAllFaces(imgEl)
    .withFaceLandmarks().withFaceDescriptors();
}

// Find the query person across the gallery
function findMatches(queryDescriptor, gallery, threshold = 0.5) {
  const matches = [];
  for (const item of gallery) {            // item = { photoId, descriptors[] }
    for (const d of item.descriptors) {
      const dist = faceapi.euclideanDistance(queryDescriptor, d);
      if (dist < (1 - threshold)) {        // smaller distance = better match
        matches.push({ photoId: item.photoId, score: 1 - dist });
        break;
      }
    }
  }
  return matches.sort((a, b) => b.score - a.score);
}
Mode A pays nothing for compute — the user's phone/laptop does the recognition. You only host a few megabytes of model files on a free static host.

11Step-by-Step Build (cheapest path)

  • 1. Prototype free (Tier 0): static React + face-api.js page, drag-drop gallery + selfie, show matches — host on GitHub Pages/Netlify for $0.
  • 2. Add a free backend (Tier 1): FastAPI + InsightFace + SQLite, deploy to Hugging Face Spaces or Render free tier.
  • 3. Storage: keep images in SQLite/disk for the MVP, or Cloudflare R2 free tier (no egress fees).
  • 4. Vectors: brute-force NumPy cosine; add FAISS only when galleries get huge.
  • 5. Frontend host: Vercel/Netlify free; point it at the backend URL.
  • 6. Domain + TLS: Cloudflare free TLS; buy a domain (~$10/yr) only when going public.
  • 7. Privacy: add consent + a "delete everything" button (one SQL wipe).
  • 8. Upgrade only on demand: move to a $5 VPS (Tier 2) when cold starts or volume hurt.

14Scaling & Free-Tier Limits

  • Cold starts: free dynos sleep — first request after idle is slow. Fix with a $5 always-on VPS or a cheap cron ping.
  • RAM limits: free tiers cap memory (often ~512 MB–1 GB); load one model and process images in small batches.
  • CPU speed: CPU inference ≈ 0.1–0.5s per image — fine for hundreds of photos; batch with a queue for thousands.
  • Brute-force search limit: NumPy cosine is fine to ~100k faces; beyond that add FAISS (still free) or pgvector.
  • Storage egress: prefer Cloudflare R2 (zero egress fees) over services that charge for downloads.

15Privacy & Security

🔒 Browser = max privacy

In Mode A photos never leave the device — the strongest privacy story and it's also the cheapest.

📜 Consent

Face data is biometric/sensitive — get explicit consent and show a clear notice.

⏳ Retention

Auto-wipe galleries after a TTL; offer one-click delete (a single SQL DELETE).

⚖️ Regulation

GDPR, BIPA (US-Illinois) and India's DPDP Act govern biometric data — build deletion in from day one.

17Monetization (with ~zero cost base)

  • Freemium: free for small galleries (covered by free tiers), paid for larger ones.
  • Pay-per-download: finding is free; charge for high-res downloads (great margins).
  • Per-event flat fee: small charge per event/gallery for photographers.
  • Tip/donation or ads: viable since infra costs nothing.
  • Upsell to hosted plan: offer a no-cold-start, larger-storage paid tier later.
